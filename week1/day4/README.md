
# Week 1 - Day 4: SQL Practice, Joins and Window Functions

## 📌 Overview

On Day 4, I continued practicing SQL with multiple datasets and different levels of queries. The work focused on strengthening SQL fundamentals, understanding joins between tables, and learning advanced analytical concepts using window functions.

## 🛠️ Tools Used

* DB Fiddle for SQL execution and practice
* GitHub for organizing and maintaining work

## 📚 Topics Covered

* SELECT and WHERE clauses
* GROUP BY and HAVING
* DISTINCT keyword
* Comparison and logical operators
* IN, NOT IN, BETWEEN, and LIKE operators
* SQL JOIN operations
* FULL OUTER JOIN concepts
* Window Functions
* ROW_NUMBER(), RANK(), DENSE_RANK()
* LAG() and LEAD() functions
* PARTITION BY and running totals

## 🗂️ Work Done

### 1. Employees Table SQL Practice

* Created and inserted employee records into the Employees table
* Solved queries related to filtering, grouping, sorting, and searching data
* Practiced operators and aggregate functions

### 2. SQL Joins Assignment

* Worked with Students, Courses, Instructors, and Enrollments tables
* Practiced LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN concepts, and CROSS JOIN
* Learned how tables are connected using foreign keys

### 3. Window Functions Assignment

* Practiced ranking functions like ROW_NUMBER(), RANK(), and DENSE_RANK()
* Calculated running totals and cumulative values
* Used LAG() and LEAD() for previous and next row analysis
* Learned the importance of PostgreSQL support for advanced SQL functions

## 📊 What I Learned Today

* How different SQL clauses work together in real-world queries
* Difference between joins and normal filtering queries
* How window functions help analyze data without grouping rows
* Importance of choosing the correct SQL engine (MySQL vs PostgreSQL)
* Better understanding of analytical SQL concepts and query structuring
